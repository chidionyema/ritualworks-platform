## Deliverable

### New files
- `src/Audit/Audit.Infrastructure/Migrations/<timestamp>_AddAuditEventsPartitioned.cs` — EF migration that runs the **raw SQL from spec § 4 verbatim** for `audit_events` + the 3 indexes + first 2 monthly partitions (current + next month). Use `migrationBuilder.Sql(...)`. Do NOT use EF's CreateTable — partitioned tables must be raw SQL.
- `src/Audit/Audit.Infrastructure/Migrations/AuditDbContextModelSnapshot.cs` — generated by `dotnet ef`; check in.
- `src/Audit/Audit.Infrastructure/Persistence/AuditWriter.cs` — implements `IAuditWriter`. COPY-batched: buffer up to 50 rows / 200ms via `System.Threading.Channels`, flush on either threshold; uses `NpgsqlConnection.BeginBinaryImport` for COPY. `IDisposable` flushes on shutdown.
- `src/Audit/Audit.Application/Capture/AuditConsumer.cs` — generic `class AuditConsumer<T> : IConsumer<T> where T : class, IDomainEvent`. Resolves `IAuditExtractor<T>`, calls `ISecretRedactor`, calls `IAuditWriter.WriteAsync`. Idempotency: read `ConsumeContext.MessageId`; on null, fall back to deterministic hash per spec.
- `src/Audit/Audit.Application/Capture/ConsumerRegistry.cs` — reflects over `Haworks.Contracts` assembly, finds every `class/record` implementing `IDomainEvent`, registers `AuditConsumer<TEvent>` with MassTransit. Used from `Audit.Api/Program.cs`.
- `tests/Audit.Integration/Audit.Integration.csproj` — new project (xUnit + Testcontainers + FluentAssertions, match `tests/Notifications.Integration`).
- `tests/Audit.Integration/AuditWebAppFactory.cs` — `WebApplicationFactory<Program>` mirroring `NotificationsWebAppFactory` — uses `SharedTestPostgres.CreateDatabaseAsync("audit")` and an in-memory or Testcontainers RabbitMQ.
- `tests/Audit.Integration/EndToEndCaptureTests.cs` — for each of these representative events, publish via MassTransit, assert one row in `audit_events` with payload matching input minus redacted fields:
  - `OrderCreatedEvent`
  - `PaymentCompletedEvent` with a `Token` property → assert redacted
  - `StockReservationFailedEvent` → assert override picks `OrderId` not `SkuId`
  - `VaultRotationStageEvent` → assert `entity_type="system"`
- `tests/Audit.Integration/IdempotencyTests.cs` — same `MessageId` published twice → exactly one row.

### Modified files
